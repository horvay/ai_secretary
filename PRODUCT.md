# AI Secretary Product Context

register: product

## Product Purpose
AI Secretary is a desktop companion app centered on Ari, a cute feminine cat-girl AI assistant. It provides conversational help, spoken responses, screenshot-aware context, memory, routines, reminders, tasks, and a visible avatar presence on the user's desktop.

## Users
Primary user is a technical, power-user desktop operator who wants an always-available companion for coding, planning, remembering, and lightweight executive-assistant style nudges. The user values speed, reliability, warmth, and direct control.

## Brand and Tone
Ari is bubbly, mischievous, nerdy, affectionate, and competent. The product should feel cozy and alive without becoming noisy or childish. The assistant can be cute, but the interface should remain functional, inspectable, and trustworthy.

## Strategic Principles
- Reliability first: the app must visibly start, respond, speak, and shut down cleanly.
- Proactive but consentful: routines and reminders need clear controls and should not surprise users with test content.
- Desktop-native: the UI should feel like a companion overlay, not a web dashboard pasted into a window.
- Inspectable automation: tool calls, routines, reminders, and memory actions should be understandable.
- Low-friction testing: CLI flags and screenshots are first-class for validating behavior.

## Anti-References
- Generic SaaS dashboards.
- Noisy debug/test copy appearing in normal user chats.
- Hidden background automations that cannot be paused or explained.
- Fragile media startup that depends on user guessing what to click.
